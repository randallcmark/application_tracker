# ADR 2026-04-19: User Context Belongs In The Account Menu

## Status

Accepted

## Decision

Persistent account-specific operations live under the logged-in username rather than the primary workflow navigation.

## Rationale

- Primary navigation should stay focused on work surfaces.
- User Settings, Capture Settings, Help, and Sign out are contextual account operations, not daily workflow destinations.
- Admin and API Docs should remain visible only for admins and grouped with related account actions.
