# ADR 2026-04-19: Navigation And Responsive Shell Are Product Infrastructure

## Status

Accepted

## Decision

The product should have a consistent navigation shell and explicit responsive behavior before additional feature-heavy surfaces are added.

## Rationale

- Users should not need to remember direct URLs between Focus, Inbox, Board, Job Workspace, Artefacts, Capture, Settings, and Admin.
- A stable home anchor should return to Focus.
- Mobile portrait usability is core product quality, not cosmetic polish.
- Board visual alignment should be completed so Active Work does not feel like a separate product.
