# ADR 2026-04-17: Use The Design System For New UI Work

## Status

Accepted

## Decision

New UI work should reference `docs/design/DESIGN_SYSTEM.md` before implementation.

## Rationale

- The product needs a consistent visual language as it evolves beyond bootstrap screens.
- The design system defines calm-precision personality, semantic color use, type scale, spacing, component states, contextual AI blocks, and external-transition conventions.
- Existing screens can migrate incrementally, but new surfaces should not introduce another style.
