# ADR 2026-04-15: Focus-Led Workspace Supersedes Board-Centred Planning

## Status

Accepted

## Decision

The board remains a supported workflow view, but it is no longer the default strategic center of the product. Planning should treat Application Tracker as a goal-aware workspace organized around Focus, Inbox, Active Work, Job Workspace, Artefacts, Capture, and Admin.

## Rationale

- A job search requires triage, preparation, follow-up, artefact reuse, external application work, and learning from outcomes.
- The UI should surface the user's goal and next useful action before raw workflow status.
- Board, lane, and list views are useful lenses over active work, but they should not define product architecture.
